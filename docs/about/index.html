About

The Ethereum Security Consortium (ESC) is a decentralized autonomous consortium where membership organizations, part-time staff, and the public work together to develop operational security standards.

The consortium was founded by Dick Olsson in July 2017 when submitting the first standard proposal as part of EIP-679.

Frequently asked questions

What is "OPSEC"?

OPSEC is an abbreviation for Operational Security â€” the process that identifies critical information or processes that need protection. In the context of Ethereum this includes operational procedures around writing secure software, such as reponsible disclosure, physical device management, LAN or WAN security etc.

How is ESC different to the Ethereum Foundation?

While the Ethereum Foundation is focusing on base layer research and development for the Ethereum platform their neutrality is important. But secure operational practices require close collaboration between organizations to ensure a responsive and consistent ecosystem in this regard. The ESC exists to close that gap.

How is ESC different to OpenZeppelin or Consensys Diligence?

ESC does not create Ethereum software or audit smart contracts. ESC is focusing on operational security practices.
